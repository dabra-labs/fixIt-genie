# FixIt Genie — Demo Script & Production Guide

**Target length:** 3:45 – 4:00
**Format:** Screen recording of phone + environment footage, voice-over or live audio
**Goal:** Show judges a real conversation — vision working, voice working, safety triggering, function calls happening live

---

## What This Demo Must Prove (Keep in Mind While Filming)

1. The agent **sees** — it identifies what's in frame without you naming it
2. The agent **speaks naturally** — it's a conversation, not a command-response loop
3. **Safety warnings trigger automatically** — before any physical action
4. **Function calls happen mid-conversation** — error code lookup, knowledge base, search
5. The experience works **hands-in-context** — you're actually doing the repair while talking

---

## Props & Setup Checklist

### Scene 1 — Car Battery
- [ ] Car hood propped open, battery clearly visible
- [ ] Battery terminals with visible corrosion (or apply baking soda + water residue 10 min before filming — it dries white and looks exactly like real corrosion)
- [ ] Good lighting — open garage door or use a work light pointed at the battery
- [ ] Phone mounted or held stable at arm's length (battery fully in frame)
- [ ] Keys OUT of ignition before filming

### Scene 2 — Washing Machine
- [ ] Washing machine with accessible control panel
- [ ] Set it to display an error code — or use tape + marker to mock "E4" on the display panel
- [ ] If your machine can actually throw an error: run it unbalanced (heavy item one side) for E4/UE
- [ ] Good front-facing light so the display is readable on camera

### Scene 3 — Electrical Panel
- [ ] Breaker box open (door swung open, breakers visible)
- [ ] One breaker visibly in the tripped position (middle/off position) — flip one yourself before filming
- [ ] Good lighting inside the panel — small flashlight taped nearby if needed
- [ ] SAFETY: You are NOT touching anything. Camera only. The agent will tell you not to anyway.

## Production Tips

### Phone Camera
- **Do NOT use a selfie cam or screen recorder alone.** Use a second camera (another phone, GoPro, or even a laptop webcam) to capture the experience — show the phone screen AND the equipment in the same frame when possible.
- If you only have one device: use screen recording with picture-in-picture of environment, OR record environment separately and edit together.
- Landscape orientation for the B-roll shots. Portrait is fine for close-ups.

### Audio
- **Use AirPods or earphones while filming.** This prevents the agent's voice from echoing back into the mic and triggering noise suppression.
- The agent's voice should be audible in the recording — turn it up before you start.
- Record in a quiet space. The audio is the demo.

### Lighting
- Natural light or a work light for the car scene. Harsh overhead is fine — you want the corrosion and terminals clearly visible.
- For the electrical panel: small flashlight or phone torch. Do not film in the dark.

### Pacing
- **Talk naturally, pause after each question.** The model needs 1-2 seconds to process your last frame before responding. Don't rush.
- If the agent interrupts you or responds before you're done — let it. That's a feature, not a bug.
- If the agent says something unexpected: adapt and respond naturally. Improvised moments make the demo feel more real.

### What To Do If The Agent Goes Off-Script
- If it asks a clarifying question you didn't expect: answer it truthfully and keep going.
- If it gives a wrong answer: stay calm, say "actually let me show you more" and move the camera. The agent will course-correct.
- If it loops or pauses: say "can you continue with the next step?" — this is a natural recovery.

### Editing
- Keep total runtime under 4 minutes.
- You can cut mid-conversation — just don't cut the safety warnings. Those are judging-relevant.
- Add subtle text overlays when the agent is doing a function call (brief pause while it searches): *"looking up knowledge base..."* or *"searching for error code..."* — makes the architecture visible.
- Color grade warm. Safety Orange matches the app's brand palette.

| Moment | Why It Matters |
|---|---|
| Agent identifies battery WITHOUT you naming it | Proves real-time vision is working |
| Safety warning fires automatically | Demonstrates safety-first architecture |

| Panel scene: agent sees the tripped breaker | Multi-domain knowledge working |
| Natural conversation flow throughout | This is what bidi-streaming actually feels like |
